Markdown Basics

Markdown is a lightweight markup language that compiles to HTML. Because the initial language specification as written by creator John Gruber is notoriously ambiguous, a variety of competing Markdown implementations and specs have arisen over the last fifteen years.

Markwright uses a modified subset of Markdown as described in CommonMark specification, as implemented by developers with Khan Academy in the simple-markdown package.

Inline elements

Links

Links are made using a combination of brackets and parens. Brackets enclose the text of the link and parens enclose the URL.

[Text content of my link](https://getmarkwright.com)

Images

Images are linked to the project with a similar syntax. Brackets enclose the <img> tag's alt attribute and parens enclose the URL.

![Alt text of my image](https://getmarkwright.com/img/logo.svg)

Note: Relative image lookups are performed from the working directory of the project, or, for unsaved documents, the user's home directory.

Emphasis

Emphasizing text uses a combination of asterisks, (optionally) underscores and tildes.

- Emphasize text using _asterisks_ or _underscores_.
- Strengthen text using **double asterisks**
- Combine with **asterisks and _underscores_**.
- Strikethrough uses ~~double tildes~~.

Block elements

Markwright supports standard Markdown block elements, as well as its own.

Headers

Create headers by prefixing lines with pound signs. The fewer pound signs, the larger the header will be.

# H1

## H2

### H3

#### H4

##### H5

###### H6

Note: In its manual pagination mode, Markwright creates a section break on an <h1>. Apart from breaking to a new page, this changes the text of the header at the top of the page to match the section title.

Quotes

Block quotations, recommended by the Chicago Manual of Style for quotations that extend beyond 100 words, can be created with by prefixing each line of the quote with a right angle bracket.

> Lorem ipsum dolor sit amet, consectetur adipiscing elit. Donec libero purus,
> mattis tempor tincidunt in, iaculis eget risus. Aliquam eget gravida tellus.
> Pellentesque efficitur orci sem, ut consectetur ipsum rhoncus in. Vivamus
> velit dolor, vehicula id mattis in, pretium sit amet urna.

Lists

Create enumerated lists with numbers and line breaks.

1. One
2. Two
3. Three

Create itemized lists with asterisks or dashes.

- Item
- Item
- Item

Fences

Preformatted text—i.e., text preserving whitespace—can be created by indenting content by four spaces, or enclosing the content in backticks. The latter can (optionally) be marked with the name of a language. Markwright will highlight the syntax of a variety of programming languages out-of-the-box.

```js
const [ b, a, r ] = "bar";
```

Blocks

Blocks are similar to fences, but used instead to indicate a logical division in content. When named, the resulting <div> element will have its class names set accordingly.

::: sidebar  
This content can be selected as `.sidebar`.  
:::

File transclusions

Content from other sources on the file system can be transcluded into your main Markwright project with the standard link syntax, prefixed with a single colon.

:[Markdown Basics](docs/markdown.md)

Note: Relative file transclusions and lookups are performed from the working directory of the project, or, for unsaved documents, the user's home directory.

Horizontal rules

Horizontal rules are made with triple dashes or asterisks, surrounded by newlines.
