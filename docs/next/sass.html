SASS Basics

SASS is a stylesheet language and preprocessor for CSS. Markwright uses the SASS language's more conventional "SCSS" syntax. A more thorough guide to SCSS is available at SASS's online guide.

If you're not familiar with CSS, you should do that before starting with SASS. The MDN web docs has a good introduction to the language.

Imports and partials

One of SASS's most compelling features is its ability to import external pieces of code. Import files relative to the project's working directory (or, for unsaved projects, the user's home directory) with the @import directive.

@import 'myFile.scss';

Variables

SASS allows for the creation of variables like a traditional programming language. Prefix identifiers with the $ sigil to indicate a variable, then assign to it like a normal CSS property. When your SASS is processed into CSS, instances of the variable will be swapped-in with the corresponding value.

$font-family: 'Helvetica Neue', 'Helvetica';

h1 {
  font-family: $font-family;
}

Because Markwright is based off a relatively recent browser rendering engine, it supports the use of both of SASS variables and CSS variables.

:root {
  --font-family: 'Helvetica Neue', 'Helvetica';
}

h1 {
  font-family: var(--font-family);
}

Should you choose to mix-and-match the two (which, personally, I would not recommend), you'll need to interpolate your SASS variable into a string in order to assign it to a CSS variable.

$font-family: 'Helvetica Neue, Helvetica';

:root {
  --font-family: #{$font-family};
}

Nesting

Nesting allows you to replicate the visual hierarchy of the DOM in your styles. This allows for significantly less repetition and more concise code.

.section .page:first-child {
  h1 {
    color: blue;
    + p::first-letter {
      font-size: 3rem;
    }
  }
}

The previous code would compile to the following CSS:

.section .page:first-child h1 {
  color: blue;
}

.section .page:first-child h1 + p::first-letter {
  font-size: 3rem;
}

While this feature is on track for eventual adoption in CSS, it's not natively available and so must be pre-processed via SASS.

Code Reuse

Mixins allow for reuse of styles, properties and selectors and are covered in extensive detail at the SASS documentation. Inheritance is also supported via the @extend directive. Again, documentation is available at the SASS website.

Math
