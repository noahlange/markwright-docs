Introduction to Styling with Markwright

The same general principles apply to using CSS to style a Markwright document and using CSS to theme a website — if you're comfortable with the latter, the former will be no sweat.

Document structure

It's important to have a good handle on the document structure before beginning to style.

div.section
  div.page
    div.header
    div.body
      div.content
        div.column
        hr.column-break
        div.column
      div.footnotes
    div.pagination

This is the basic hierarchy of divs in a Markwright document. A document consists of one or more sections, which contains one or more pages. A page consists of three stacked divs, one for the header, another for pagination and the main body of the page sandwiched between the two.

The page body consists of content (i.e., your body text) and a container for your footnotes. Content itself will be set in one or more columns within the page body.

Recipes

Selecting specific sections

Markwright will automatically add the section number as a second class on each section's classList. They can be selected accordingly.

.section-1 .page {
  background-image: url('images/backgrounds/section-1.png');
}

Selecting specific pages

Markwright will automatically add the page number as a second class on each page's classList. They can be selected accordingly.

.page-3 {
  background-color: black;
}

Hiding section headers or pagination

For a less-documenty document—posters, &c.—you may wish to hide the header and pagination. This should be done by setting the visiblity property of each div to hidden. You may run into positioning trouble if you use display: none;, which will remove the elements from the DOM instead of preserving their dimensions and hiding the content.

.page-1 {
  .header, .pagination {
    visibility: hidden;
  }
}

Resizing document columns

You may wish to adjust the relative width of columns. This can be done with the first-child, last-child and nth-of-type pseudo-selectors.

Columns are laid out as a single row in the .body div using flexbox, so you'll need to adjust columns' flex-basis instead of their width. Here's a basic mixin to set the widths of n columns:

@mixin column-widths($widths...) {
  @for $i from 1 through length($widths) {
    .column:nth-of-type(#{$i}) {
      flex-basis: nth($widths, $i);
    }
  }
}

.page-1 {
  // for a three-column layout
  @include column-widths(2in, 3in, 1in);
}

Selecting images

Images can be selected with attribute selectors.

img[src$="my-picture.png"] {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  z-index: 0;
}
